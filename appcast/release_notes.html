Version 2.1.3

Bugfixes

  • Fixes a bug that would limit the HSNE Random Walks Threshold parameter range to [0.0..1.0] even though the default value used is 1.5.

Version 2.1.2

New Features

  • If one of the name fields in fcs is not set but the other is the name will be copied to the empty one.
  • Sets NaN values to zero on load (a warning will be printed in the console window).

Version 2.1.1

New Features

  • Data Level HSNE Coordinates are now added to exported fcs files if available. If not zoomed into the data level coordinates will be (0, 0) for the corresponding points. There is no indication of the map the coordinates belong to within the files, make sure to aptly name your HSNE analyses and exported files to keep this information.
  • It is now possible to save clusters while maintaining the original sample structure. I.e. assume you have two samples as separate fcs files and 3 resulting clusters
    • Using Save Clusters ... saves them as cluster1.fcs, cluster2.fcs and cluster3.fcs as before
    • Using Save Clusters (per sample)... saves them as cluster1-Sample1.fcs, cluster1-Sample2.fcs, cluster2-Sample1.fcs and and so on.
  • Added a confirmation dialog on exit.

Version 2.0.4

New Features

  • t-SNE/HSNE: The scatterplot view can now be recorded as video. Toggle the record switch (below "dark mode") to turn recording on and off and once done save a video through in the screen shot dialog (camera icon).
  • FCS Reader: now allows loading files with different markers as long as the number of markers is the same. Different markers will be represented as -multiple- in the marker list and should be excluded from the computation.

Bugfixes

  • t-SNE/HSNE: Removes export clusters from scatterplot context menu, when clusters are not up to date.
  • FCS Reader: now gives the correct error if the number of markers does not match.
  • General: Updates the about dialog.
  • General: Changes contact email to license@cytosplore.org
